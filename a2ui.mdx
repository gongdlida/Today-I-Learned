# A2UI 학습 정리

## 1. A2UI를 처음 접하면 생기는 오해

A2UI를 처음 접했을 때 흔히 드는 생각은 다음과 같다.

- “LLM으로 UI를 자동 생성하는 기술인가?”
- “자연어로 화면을 바로 만들어주는 도구인가?”

하지만 A2UI는 아래와 같은 방향과는 다르다.

실시간 운영 UI 엔진, UI 코드 생성기가 아닌
**결정 과정**에 집중하는 개념이다.

---

## 2. A2UI가 던지는 핵심 질문

A2UI의 출발점은 다음 질문이다.

LLM을 UI를 그리는 도구가 아니라,  
UI를 *결정*하는 도구로 사용할 수 있을까?

즉, “어떤 컴포넌트를 렌더링할 것인가”가 아니라  
“왜 이 UI가 필요한가, 무엇이 중요한가”를 구조화하는 데 목적이 있다.

---

## 3. A2UI의 기본 동작 원리

A2UI는 다음과 같은 파이프라인을 전제로 한다.

```
자연어 입력
→ 의도 해석 (Intent Parsing)
→ UI 의미 구조(IR, JSON)
→ 스키마 / 디자인 시스템 제약
→ 팀 전용 렌더러
```

핵심은 **LLM의 책임을 명확히 제한**하는 것이다.

- LLM: 의미와 의도 해석
- 시스템: 규칙과 제약
- 렌더러: 실제 UI 표현

---

## 4. UI 중간 표현(IR)이 중요한 이유

A2UI에서는 UI를 시각 요소로 직접 다루지 않는다. (버튼, 카드, 레이아웃)

- 의도(intent), 역할(role), 우선순위(priority), 맥락(context)

예시 (개념적):

```json
{
  "intent": "primary_action",
  "context": "checkout_complete",
  "priority": "high"
}
```

이 단계에는 JSX, 스타일, 색상 정보가 없다.  
UI를 **의미 단위의 모델**로 다루기 위함이다.

---

## 5. 일반적인 LLM UI 생성 방식과의 차이

| 구분          | 일반 LLM UI 생성 | A2UI               |
| ------------- | ---------------- | ------------------ |
| 출력          | JSX / HTML       | 의미 중심 JSON(IR) |
| 초점          | 시각적 결과      | UX 결정 구조       |
| 디자인 시스템 | 추측             | 전제 조건          |
| 검증 가능성   | 낮음             | 높음               |
| 실무 안정성   | 낮음             | 상대적으로 높음    |

A2UI는 “코드를 생성하지 않기 때문에”  
오히려 디자인 시스템과 함께 실무에 연결되기 쉽다.

---

## 6. A2UI는 디자인 시스템을 학습시키는가?

아니다.

A2UI에서 디자인 시스템은:

- AI가 외워야 할 대상이 아니라
- **AI가 반드시 따라야 할 제약 조건**이다.

권장 구조는 다음과 같다.

- LLM: 자연어 → intent
- Mapper(코드): intent → 팀 타입 / variant
- Renderer: 디자인 시스템 기반 컴포넌트 출력

디자인 시스템은 항상 **Source of Truth**로 남아야 한다.

---

## 7. JSON 구조가 팀 타입과 맞지 않을 때

이 경우에도 전체 디자인 시스템을 학습시키는 것은 피해야 한다.

학습 가능한 영역:

- 자연어 → 팀 내부 intent 체계

학습하면 안 되는 영역:

- 컴포넌트 타입
- variant 이름
- 스타일 규칙

AI는 규칙을 아는 존재가 아니라,  
**사람의 말을 팀의 언어로 번역하는 역할**에 가깝다.

---

## 8. A2UI는 렌더러를 제공하는가?

A2UI는 렌더러를 제공하지 않는다.

A2UI가 다루는 것은:

- “무엇을 보여줄지”에 대한 결정

렌더러가 다루는 것은:

- “어떻게 표현할지”에 대한 구현

이 분리 덕분에 React, Native, Web, TV 등  
플랫폼에 종속되지 않는 구조가 가능해진다.

---

## 9. A2UI는 언제 쓰는 게 적절한가?

부적합한 경우:

- 사용자별 실시간 자동 UI 변경
- 운영 환경에서의 완전 자동화

적합한 경우:

- 개발 단계 UI 초안 생성
- UX 패턴 추천
- Server-Driven UI 스키마 설계 보조
- Story 조합 제안 (사람 승인 포함)

A2UI는 운영 엔진이 아니라 **개발 보조 도구**에 가깝다.

---

## 11. 한 문장 요약

A2UI는 AI로 UI를 만드는 기술이 아니라,  
UX 의사결정을 구조화하고 가속하기 위한 사고 모델이다.
